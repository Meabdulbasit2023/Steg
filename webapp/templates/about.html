Algorithm 1 — Design

The hiding pipeline

Secret→ Binary→ 2's complement→ XOR→ ML region selection→ Embed→ Stego image
  1. Binary conversion — the secret (image or text) becomes a stream of bytes, then bits.
  2. Two's complement — every byte is transformed: S₂comp = NOT(S) + 1. The transform is its own inverse, so applying it again during decoding restores the original bytes.
  3. XOR — each bit is XORed with the most-significant bit of the pixel it will occupy, scrambling the data against the image itself. Without reversing both steps, an extracted bit-stream is meaningless.
  4. ML region selection — a Random Forest model trained on the project dataset classifies every 8×8 block by texture (variance + edge strength) as busy, medium or smooth. Bits go into the busiest blocks first, where changes are imperceptible.
  5. Embedding — the transformed bits replace the lowest 1–2 bits of the chosen pixels. A small header (marker, payload type, bits-per-channel, length) is stored at a fixed location so the decoder can bootstrap itself.

Worked example

Binary → two's complement, by hand

Hiding the letter A (ASCII 65):

stepoperationresult
1'A' as decimal65
2convert to binary 01000001
31's complement — flip every bit (NOT) 10111110
4add 1 → 2's complement 10111111 (= 191)
5each bit ⊕ pixel MSB, written to pixel's low bitembedded

Decoding reverses it: read bits → ⊕ MSB again → 10111111 → NOT+1 → 01000001 = 65 = A. The encode page shows this exact table computed live for your own secret.

Machine learning

The two ML models

1 · Random Forest region model

Trained on 8×8-block texture features (82,396 blocks from 198 passport photos + 10 varied benchmarks) to classify blocks as busy, medium or smooth. At embed time it ranks each block of the specific cover image; the ranking is computed from bit-masked pixels so encoder and decoder always derive identical regions — no key exchange needed.

2 · SVM steganalysis classifier

A trained "attacker" (support vector machine, RBF kernel) that tries to detect hidden data from LSB-plane statistics. Detectability near 50% means stego images are statistically indistinguishable from clean ones. The evaluation compares our method against traditional sequential LSB embedding.

Objective iii

Evaluation metrics

  • MSE — mean of (original − stego)² over all pixels.
  • PSNR — 10·log₁₀(255² / MSE), target > 40 dB.
  • Extraction accuracy — % of payload bytes recovered exactly (100% by design on lossless PNG).
  • Detectability — accuracy of the trained steganalysis attacker (closer to 50% is better).
